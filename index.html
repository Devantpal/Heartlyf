About the Project

Cardiac safety powered by edge AI

Cardiovascular diseases are the world's leading cause of death. Heartlyf is a final-year IoT project from GNIOT that addresses the critical gap in continuous, remote cardiac monitoring — combining wearable hardware, machine learning and cloud infrastructure into one cohesive system.

🧠
TinyML Arrhythmia Detection
CNN and Random Forest models trained on the MIT-BIH arrhythmia dataset, converted to TensorFlow Lite and deployed directly on the ESP32 for real-time offline classification.
📡
Real-Time Firebase Cloud
Firebase Realtime Database stores every ECG sample with millisecond precision. Cloud Functions trigger instantly on anomalies to run secondary detection algorithms.
🚨
GPS-Enabled Emergency Response
When a critical event is confirmed, the system simultaneously fires Telegram messages, Twilio SMS, auto-call and a live Google Maps link to all registered emergency contacts.

🛠️ Hardware Layer

ESP32 DevKit (dual-core 240 MHz), AD8232 analog front-end, MAX30105 optical sensor, NEO-6M GPS module and LiPo battery with power management circuit.

🔬 Signal Processing

5th-order Butterworth high-pass filter (0.5 Hz cutoff) removes baseline wander. Chebyshev low-pass (40 Hz) eliminates EMG noise. Pan-Tompkins algorithm detects QRS peaks.

🧠 AI Classification

CNN model classifies 5 rhythm classes: Normal, Atrial Fibrillation, Ventricular Tachycardia, PVC, and Bundle Branch Block — all running on-device at 200ms inference time.

Methodology

How Heartlyf works

01

Signal Acquisition

The AD8232 ECG sensor uses a 3-lead electrode system (RA, LA, RL) to capture the heart's electrical activity. The MAX30105 simultaneously measures blood oxygen saturation and pulse rate — providing two independent vital signs for higher diagnostic confidence.

02

On-Device Preprocessing & AI

The ESP32 samples the ADC at 360 Hz, applies Butterworth and Chebyshev digital filters to remove noise and baseline wander, extracts RR-intervals, and runs the quantized TinyML model to classify the current rhythm in approximately 200ms — entirely offline.

03

Cloud Transmission & Storage

Clean, classified data packets are transmitted via Wi-Fi to Firebase Realtime Database every 500ms. When offline, the SD card module logs all sessions locally and syncs automatically when connectivity is restored — ensuring zero data loss.

04

Cloud Analysis & Threshold Checking

Firebase Cloud Functions trigger on every database write. A secondary anomaly detection layer checks BPM against configurable thresholds (Tachycardia >100 BPM, Bradycardia <50 BPM), SpO2 drop, and cross-validates with the edge AI classification result.

05

Emergency Notification Dispatch

Upon confirmed anomaly, the system simultaneously sends a Telegram bot message, a Twilio SMS, an automated phone call, and a Google Maps link with the patient's real-time GPS coordinates — reaching family, caretakers and doctors in under one second.
